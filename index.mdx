Mise (mise-en-place) is an all-in-one developer environment manager. It replaces version managers like `nvm`, `pyenv`, and `rbenv`; environment tools like `direnv`; and task runners like `make` — all with a single, fast Rust binary.

## What mise does

<CardGroup cols={3}>
  <Card title="Tool Version Manager" icon="code-branch">

    Like direnv — automatically load environment variables when you enter a project directory.

  <Card title="Task Runner" icon="play">
    Like make — define tasks in `mise.toml` or as shell scripts, with parallel builds and file watching.

## Quick example

```toml mise.toml
[tools]
node = "22"
python = "3.12"
go = "1.22"

[env]
NODE_ENV = "development"
DATABASE_URL = "postgres://localhost/myapp"

[tasks.build]
run = "cargo build"

[tasks.test]
run = "cargo test"
depends = ["build"]
```

With this single file, mise will:
- Install and activate the right versions of Node, Python, and Go
- Set your environment variables when you enter the directory
- Provide `mise run build` and `mise run test` commands with dependency ordering

## Get started

<Steps>
  <Step title="Install mise">
    ```bash
    curl https://mise.run | sh
    ```
  </Step>
  <Step title="Activate in your shell">
    ```bash
    # Add to ~/.zshrc, ~/.bashrc, or ~/.config/fish/config.fish
    eval "$(~/.local/bin/mise activate zsh)"
    ```
  </Step>
  <Step title="Add tools to your project">
    ```bash
    mise use node@22 python@3.12
    ```
  </Step>
  <Step title="Run your tasks">
    ```bash
    mise run build
    mise run test
    ```
  </Step>
</Steps>
